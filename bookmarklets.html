Bookmarklets

Bookmarklets sind Lesezeichen, die ein kleines Script ausführen. Damit kannst du SafeShare auf Desktop sehr schnell nutzen: aktuelle URL übernehmen → SafeShare öffnen → optional direkt bereinigen.

App öffnen Hilfe Pro ansehen
iOS/iPad: Bookmarklets sind je nach Safari-Version eingeschränkt. Für „echtes 1-Klick“ auf iOS ist ein Shortcut im Share-Sheet stabiler.

1) App öffnen & URL nur ausfüllen

Übernimmt die aktuelle Seite in SafeShare (Parameter u). Du klickst danach manuell auf „Bereinigen“.

Bookmarklet

javascript:(function(){try{var u=location.href;location.href='https://j-ai-71.github.io/Supersystem/app.html?u='+encodeURIComponent(u);}catch(e){alert('SafeShare: konnte URL nicht lesen.');}})();
Technisch: Dieses Bookmarklet nutzt eine absolute URL, damit es auf jeder Website funktioniert (nicht nur innerhalb von SafeShare).

2) App öffnen & sofort bereinigen (1-Klick)

Wie oben, aber mit run=1. SafeShare übernimmt die URL und startet direkt die Bereinigung.

Bookmarklet

javascript:(function(){try{var u=location.href;location.href='https://j-ai-71.github.io/Supersystem/app.html?u='+encodeURIComponent(u)+'&run=1';}catch(e){alert('SafeShare: konnte URL nicht lesen.');}})();
Voraussetzung: app.html muss ?u= lesen und bei run=1 automatisch bereinigen.

3) „Einfügen“-Bookmarklet (wenn du nicht auf der Zielseite bist)

Fragt dich nach einer URL (einfügen) und öffnet SafeShare direkt mit Auto-Bereinigung.

Bookmarklet

javascript:(function(){try{var u=prompt('SafeShare: URL einfügen');if(!u)return;location.href='https://j-ai-71.github.io/Supersystem/app.html?u='+encodeURIComponent(u)+'&run=1';}catch(e){alert('SafeShare: konnte URL nicht übernehmen.');}})();
Hinweis: Das ist kein Clipboard-Zugriff. Du fügst manuell ein (Browser-Sicherheit).

Einrichten (Desktop)

  1. Lesezeichen erstellen (Bookmark hinzufügen)
  2. Name: z. B. SafeShare / SafeShare 1-Klick
  3. URL/Feld: den Bookmarklet-Code einfügen (muss mit javascript: beginnen)
  4. Auf einer Website klicken → SafeShare öffnet
Tipp: Viele Browser entfernen Leerzeichen/Zeilenumbrüche automatisch – trotzdem am besten den Code genau als eine Zeile einfügen.

Fehlersuche (wenn es „nicht geht“)

Es passiert nichts

  • Prüfe: beginnt wirklich mit javascript:?
  • Wurde der Code vom Browser „bereinigt“/gekürzt? Dann neu einfügen.
  • In manchen Setups blockt der Browser Bookmarklets aus Sicherheitsgründen → Shortcut nutzen.

SafeShare öffnet, aber Feld bleibt leer

  • Dann übernimmt app.html den Parameter u noch nicht.
  • Lösung: Patch in app.html (Übernahme von ?u= + optional run=1).
iOS/iPad: Wenn Bookmarklets zicken, nutze den Share-Sheet Shortcut (robust) statt Browser-Bookmarklet.
