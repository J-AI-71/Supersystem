Bookmarklets

SafeShare mit 1 Klick öffnen – und die aktuelle URL übernehmen

Ein Bookmarklet ist ein normales Lesezeichen, das ein kleines Script im aktuellen Tab ausführt. Damit kannst du die URL, die du gerade ansiehst, direkt an SafeShare übergeben – ohne Copy/Paste.

Bookmarklets Einrichten Fehlersuche
iOS/Safari und „In-App-Browser“ (z. B. Instagram/TikTok) schränken Bookmarklets oft ein. Wenn es hakt: SafeShare App öffnen → Link einfügen → bereinigen.

Wie es technisch funktioniert (kurz)

Übergabe per ?u=

Das Bookmarklet liest location.href (die aktuelle URL) und öffnet danach SafeShare mit ?u=…. Dadurch kann SafeShare die URL automatisch ins Eingabefeld übernehmen.

Damit es „1-Klick“ ist, sollte app.html den Parameter u beim Laden übernehmen. (Wenn das bei dir noch fehlt: sag Bescheid – ich gebe dir den minimalen Patch für app.html.)

Sicherheits-/Browser-Limits

Bookmarklets dürfen nur sehr begrenzt arbeiten: sie können i. d. R. nur die aktuelle URL lesen und umleiten. Viele Browser blockieren Bookmarklets in eingebetteten Browsern oder aus Sicherheitsgründen komplett.

Wenn ein Browser Bookmarklets blockiert, ist das kein „Bug“ von SafeShare – sondern eine Plattform-Beschränkung.

Bookmarklets (fertig)

Empfehlung: Nutze die absolute Variante. Sie ist unabhängig von Pfaden und verhindert viele 404/Fehlleitungen.

1) SafeShare App öffnen (empfohlen, absolute URL)

Öffnet SafeShare App und übergibt die aktuelle URL.

javascript:(function(){try{var u=location.href;location.href='https://j-ai-71.github.io/Supersystem/app.html?u='+encodeURIComponent(u);}catch(e){alert('SafeShare: konnte URL nicht lesen.');}})();
Test: URL-Übernahme
Test-Link öffnet SafeShare mit example.com. Wenn das Eingabefeld leer bleibt, fehlt in app.html das Lesen von ?u=.

2) Pro-Seite öffnen (nur Landing, optional)

Öffnet die öffentliche Pro-Übersicht und übergibt die aktuelle URL (für Kontext/Copy-Transfer).

javascript:(function(){try{var u=location.href;location.href='https://j-ai-71.github.io/Supersystem/pro.html?u='+encodeURIComponent(u);}catch(e){alert('SafeShare Pro: konnte URL nicht lesen.');}})();
Die gekaufte Pro-App liegt nicht öffentlich. Für Käufer ist der schnellste Einstieg: Payhip-Download → 01-START.html → dort Pro-App öffnen.

Einrichten (Desktop)

  1. Lege ein neues Lesezeichen an (z. B. „SafeShare Clean“).
  2. Bearbeite das Lesezeichen.
  3. Füge den Bookmarklet-Code in das URL-Feld ein (beginnt mit javascript:).
  4. Beim Surfen: Lesezeichen anklicken → SafeShare öffnet mit der aktuellen URL.
Manche Browser entfernen javascript:, wenn du es „zu bequem“ einfügst. Dann: erst speichern → danach nochmal bearbeiten → Code erneut in das URL-Feld.

iPad/iPhone: was realistisch funktioniert

Stabiler Standard-Workflow

  • Link kopieren
  • SafeShare App öffnen
  • Einfügen → bereinigen → Ergebnis teilen
Das ist auf iOS oft schneller als „Bookmarklet-Troubleshooting“.

Warum Bookmarklets auf iOS oft zicken

  • Safari blockiert/limitiert JavaScript-Bookmarks je nach Version und Kontext.
  • In-App-Browser (Social Apps) sind häufig noch strenger.
  • Service Worker/Cache kann Updates „kleben“ lassen → siehe Hilfe.
Wenn du 1-Klick auf iOS willst, ist ein Shortcut im Share-Sheet meist die robustere Lösung als Bookmarklets.

Fehlersuche (Checkliste)

A) Klick macht „nichts“

  • Passiert es in einer App (In-App-Browser)? → dort sind Bookmarklets oft blockiert.
  • Teste im normalen Safari/Chrome Desktop auf einer Website.
  • Fallback: Copy/Paste in app.html.

B) SafeShare öffnet, aber URL fehlt

  • Öffne den Test-Link oben („URL-Übernahme“).
  • Wenn das Eingabefeld leer bleibt: app.html übernimmt ?u= noch nicht.
  • Dann ist es kein Bookmarklet-Problem, sondern ein kleiner App-Hook.

C) Du landest auf einer falschen Seite / 404

  • Nutze die absolute Bookmarklet-Variante (empfohlen).
  • Relative Links können je nach Pfad falsch auflösen.

D) Copy-Button funktioniert nicht

  • Clipboard kann blockiert sein (Kontext/Browser/iOS).
  • Dann: Code-Block markieren → manuell kopieren.
Wenn du mir zur Diagnose schreibst: Gerät, Browser und ob es in Safari direkt oder in einer App passiert, ist die Ursache in der Regel sofort klar.
