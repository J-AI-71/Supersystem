Bookmarklets

Bookmarklets sind Lesezeichen, die JavaScript im aktuellen Tab ausführen. Damit kannst du die aktuelle Seiten-URL in SafeShare übernehmen – ohne Copy/Paste.

App öffnen Pro Fehlersuche
iOS/Safari schränkt Bookmarklets ein. Wenn es zickt: SafeShare App + Copy/Paste ist der zuverlässige Fallback.

Voraussetzungen (damit es wirklich „1-Klick“ ist)

App muss ?u= lesen

Dieses Bookmarklet übergibt die aktuelle URL als Query-Parameter u. Damit du nicht nochmal einfügen musst, sollte app.html (und optional Pro) beim Laden u übernehmen.

Wenn du willst: ich gebe dir direkt den minimalen JS-Patch für app.html/pro.html (Query lesen → Input füllen → optional auto-clean).

Cross-Origin Limit (ist normal)

Ein Bookmarklet kann nur location.href der aktuellen Seite lesen. Inhalte anderer Tabs, Clipboard, oder eingebettete „In-App Browser“ sind oft eingeschränkt.

In Apps (Instagram/TikTok/Facebook In-App Browser) sind Bookmarklets häufig blockiert → Copy/Paste.

1) SafeShare App öffnen (mit aktueller URL)

Öffnet SafeShare App und übergibt die aktuelle URL als u.

Bookmarklet (relativ)

Funktioniert zuverlässig, wenn du das Bookmarklet von einer Seite aufrufst – und anschließend in SafeShare landest. Diese Variante nutzt location.href='app.html?...'.

javascript:(function(){try{var u=location.href;location.href='app.html?u='+encodeURIComponent(u);}catch(e){alert('SafeShare: konnte URL nicht lesen.');}})();
Robustere Alternative (absolute URL) siehe weiter unten – nützlich, wenn relative Pfade falsch auflösen.

2) SafeShare App öffnen (absolute URL)

Diese Variante ist in der Praxis stabiler, weil sie unabhängig vom aktuellen Pfad immer auf deine GitHub-Pages-URL springt.

Bookmarklet (empfohlen)

javascript:(function(){try{var u=location.href;location.href='https://j-ai-71.github.io/Supersystem/app.html?u='+encodeURIComponent(u);}catch(e){alert('SafeShare: konnte URL nicht lesen.');}})();
Das ist die beste Default-Variante für Desktop-Bookmarks (weniger „Pfad-Überraschungen“).

3) Pro öffnen (Hinweis)

Achtung: Deine pro.html ist öffentlich (Landing/Preise). Die Pro-App selbst liegt nicht öffentlich (Payhip-Download → 01-START.html).

Ein Bookmarklet kann keine lokale Datei (file://) zuverlässig „anspringen“ und auch nicht auf den Payhip-Download „hineinlinken“, ohne dass der Nutzer die Datei bereits lokal geöffnet hat.

Bookmarklet: Pro-Landing öffnen (mit URL)

javascript:(function(){try{var u=location.href;location.href='https://j-ai-71.github.io/Supersystem/pro.html?u='+encodeURIComponent(u);}catch(e){alert('SafeShare Pro: konnte URL nicht lesen.');}})();
Für Käufer ist der beste „1-Klick“-Weg: 01-START.html als Startseite im Browser speichern/als Favorit – dort Pro-App öffnen.

Einrichten (Desktop, Chrome/Firefox/Safari macOS)

  1. Neues Lesezeichen erstellen
  2. Name: z. B. „SafeShare Clean“
  3. URL/Feld: kompletten Code einfügen (beginnt mit javascript:)
  4. Beim Surfen anklicken → SafeShare öffnet
Manche Browser löschen javascript: aus Sicherheitsgründen, wenn du es in der UI „normal“ einfügst. Dann hilft: Lesezeichen anlegen → bearbeiten → Code in das URL-Feld pasten.

Fehlersuche (Checkliste)

A) Nichts passiert beim Klick

  • Browser blockiert Bookmarklets (häufig iOS / In-App Browser).
  • Test: Öffne eine normale Website im Safari (nicht in einer App) → probiere dort.
  • Fallback: Copy/Paste in app.html.

B) SafeShare öffnet, aber URL ist nicht übernommen

  • App liest ?u= noch nicht → dann bleibt das Feld leer.
  • Test: öffne direkt: app.html?u=https%3A%2F%2Fexample.com – sollte ins Eingabefeld wandern.
  • Wenn nicht: Patch in app.html nötig (Query-Param übernehmen).

C) Du landest auf einer falschen URL / 404

  • Nutze die absolute Bookmarklet-Variante (empfohlen).
  • Wenn du auf Subpages/Unterordnern bist, kann app.html relativ falsch auflösen.

D) „Kopieren“ klappt nicht

  • Clipboard API ist in manchen Kontexten gesperrt (HTTP, In-App, ältere iOS).
  • Fallback: manuell markieren (im Code-Block) und kopieren.
Debug-Signal: Wenn du mir sagst Browser + Gerät und ob es im Safari direkt oder in einer App passiert, kann man sofort einordnen, ob es ein „Bookmarklet-Limit“ ist oder ein App-Bug (?u).

iPad/iPhone: empfohlener Workflow

  • Stabil: Link kopieren → in App einfügen → bereinigen → teilen
  • Wenn du „1-Klick“ willst: iOS Shortcut im Share Sheet ist zuverlässiger als Bookmarklets
Bookmarklets sind auf iOS nicht kaputt „bei dir“, sondern systembedingt oft eingeschränkt.
