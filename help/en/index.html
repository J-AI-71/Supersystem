Help

SafeShare cleans tracking parameters from links locally in your browser. No accounts. No uploads.

Open the App Privacy Contact
This is link hygiene. It does not make you “invisible” on the web (websites still see requests, IP, user agent, cookies, etc.).

Quick start (20 seconds)

  1. Copy a link (from your browser, chat, email, social app)
  2. Open SafeShare App → paste the URL
  3. Tap Clean → then Copy or Share

Before

https://example.com/page?utm_source=x&utm_campaign=y&fbclid=XYZ

After

https://example.com/page

Common issues

“Paste” doesn’t insert anything (iOS / browser)

Some browsers require user interaction or explicit permission for clipboard access. Tap inside the field and use the system paste menu.

Share button is missing / doesn’t open

Sharing depends on your browser. If navigator.share is not available, SafeShare will fall back to copying.

A cleaned link doesn’t work

Some parameters are functional (not tracking). Try:

  • Open the original link first (to verify it works)
  • Then open the cleaned link
  • If only the cleaned link fails, a parameter was likely required → use a more conservative approach or Pro (audit/allowlist)
Don’t remove everything blindly: affiliate keys like ref or tag can control attribution/revenue.

What SafeShare does (and doesn’t) do

Does

  • Removes common tracking keys (e.g. utm_*, gclid, fbclid)
  • Runs local-first (in your browser)
  • Lets you copy/share cleaned links

Does not

  • Hide your IP from the destination website
  • Block cookies or fingerprinting on the destination website
  • Undo redirects that add IDs again after opening
SafeShare focuses on a concrete step: removing tracking parameters before you share a link.

Pro vs App (short)

The free web app is intentionally conservative: remove common tracking keys, keep functional parameters. Pro is the advanced mode for power users and publishers: policies, allowlists, and audit.
