Help

Quick fixes and practical notes for SafeShare.

Open the App Privacy Start

1) “Nothing changes” / old version still shows

  • Hard reload (Safari): reload icon → long press → Reload Without Content Blockers (or just reload twice).
  • If you use a service worker/PWA: uninstall the Home Screen app and open in Safari once, then reinstall.
  • Make sure your shell file names match exactly (including version) and do not 404.
If /css/ss-shell.2026-01-24-04.css or /js/ss-shell.2026-01-24-04.js returns 404, the nav will not render.

2) Clipboard paste fails on iOS/iPadOS

  • Safari may block programmatic clipboard reads. Tap into the field and use iOS “Paste”.
  • Alternatively: copy the link again, then open SafeShare and paste.

3) Cleaned link does not work

  • Some parameters are functional (language, product variants, affiliate tags).
  • Try opening the original link and compare.
  • Use SafeShare Pro for audit/allowlists/policies (keep selected parameters).
