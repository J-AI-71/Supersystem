2. Grundprinzip von SafeShare (local-first)

SafeShare ist eine Web-App zur Link-Hygiene. Die Bereinigung von Links erfolgt nach Möglichkeit lokal im Browser. Das bedeutet: Der eingegebene Link muss nicht an einen Server übertragen werden, damit die Bereinigung funktioniert (sofern du keine externen Dienste nutzt).

Ziel: Datensparsamkeit. Keine unnötige Weitergabe von Links/Parametern an Dritte für die Kernfunktion.

3. Hosting / Zugriffsdaten

Beim Aufruf dieser Website verarbeitet der Hosting-Provider technische Zugriffsdaten (z. B. IP-Adresse, Zeitpunkt, angeforderte Datei, User-Agent), um die Website auszuliefern und die Sicherheit zu gewährleisten.

Rechtsgrundlage ist Art. 6 Abs. 1 lit. f DSGVO (berechtigtes Interesse an Betrieb und Sicherheit).

Hinweis: Diese Website wird über GitHub Pages bereitgestellt. Wenn du zusätzlich Netlify/andere Dienste verwendest, ergänze hier den jeweiligen Anbieter und ggf. Angaben zu Logfiles/Speicherdauer.

4. Service Worker / Offline-Cache

SafeShare kann einen Service Worker einsetzen, um Seiten/Assets im Browser zwischenzuspeichern (Cache), damit die Website schneller lädt und offline robuster ist. Diese Speicherung passiert auf deinem Gerät.

  • Speicherort: Browser-Cache (Cache Storage)
  • Zweck: Performance, Offline-Funktion, Stabilität
  • Du kannst den Cache in status.html löschen bzw. den Service Worker entfernen.

5. Kontaktaufnahme

Wenn du per E-Mail Kontakt aufnimmst, verarbeiten wir die von dir übermittelten Daten (z. B. E-Mail-Adresse, Inhalt), um dein Anliegen zu bearbeiten.

Rechtsgrundlage ist Art. 6 Abs. 1 lit. b DSGVO (vorvertraglich/Vertrag) oder Art. 6 Abs. 1 lit. f DSGVO.

6. Externe Links / Drittanbieter (z. B. Payhip)

Auf dieser Website können Links zu externen Anbietern enthalten sein (z. B. Payhip für den Kauf von SafeShare Pro). Wenn du einem externen Link folgst, verlässt du diese Website. Ab dann gelten die Datenschutzregeln des jeweiligen Anbieters.

7. Affiliate-/Referral-Links (falls genutzt)

Wenn Affiliate-Links verwendet werden, kann der Zielanbieter Parameter zur Zuordnung (Attribution) nutzen. SafeShare entfernt Tracking-Parameter standardmäßig häufig – bei Affiliate-/Partnerlinks kann das die Zuordnung beeinflussen. In SafeShare Pro können Regeln/Whitelists genutzt werden, um bewusst zu steuern, was erhalten bleiben soll.

8. Cookies / Tracking

SafeShare setzt für die Kernfunktion grundsätzlich kein serverseitiges Tracking voraus. Falls zukünftig Analyse-/Tracking-Tools eingesetzt werden, muss dieser Abschnitt erweitert werden (inkl. Einwilligung/Consent, Anbieter, Zweck, Rechtsgrundlage, Speicherdauer).
