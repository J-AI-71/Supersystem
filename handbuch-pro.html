Handbuch: SafeShare Pro

SafeShare Pro ist für Workflows mit vielen Links: Bulk → Audit → Export. Der Fokus ist deterministisch (gleiches Ergebnis bei gleicher Policy) und nachvollziehbar (jede Änderung erklärbar).

Kurzstart: Textblock einfügen → Bereinigen → Audit prüfen (OK/WARN/RISK) → Export CSV/JSON.

Inhalt

1) Profile 2) Audit: OK / WARN / RISK 3) Whitelist-Fix (Signatur/Token) 4) Policy Editor: Whitelist & Blacklist 5) Domain Overrides 6) Redirect/Wrapper entpacken 7) Allowlist-First (Strict) 8) Exporte: Output.txt + Audit CSV/JSON + Policy JSON 9) FAQ: Link kaputt?

1) Profile

Profile setzen Defaults. Du kannst danach alles manuell anpassen.

  • Journalismus/Redaktion: Tracking + Affiliate entfernen, Unwrap an, Guardrails strikt.
  • Social/Creator: Tracking entfernen, Affiliate standardmäßig aus (Toggle), schnell & kanonisch.
  • Publisher/Affiliate: Tracking entfernen, Affiliate bleibt (fair).
  • Team/Compliance: audit-lastig, Unwrap an, optional „Allowlist-First“.
Profi-Praxis: Tracking (UTM/click-IDs) meist raus. Tokens/Signaturen niemals blind entfernen → Whitelist/Fix nutzen.

2) Audit: OK / WARN / RISK

Das Audit zeigt pro Link, was passiert ist (Original → Clean, Parameter-Entscheidungen, Warnungen).

  • OK: keine riskanten Eingriffe erkannt.
  • WARN: Hinweise (z. B. Wrapper nicht offline entpackbar).
  • RISK: es wurden signatur-/token-ähnliche Keys entfernt (Link könnte brechen).
Beispiele für RISK-Keys: token, sig, signature, hmac, expires, state, nonce, X-Amz-*

3) Whitelist-Fix

Wenn SafeShare Pro RISK meldet, kannst du per Whitelist-Fix die betroffenen Keys automatisch zur Whitelist hinzufügen und direkt neu bereinigen.

  • Nutze Fix, wenn ein Link nach dem Cleaning nicht mehr funktioniert.
  • Whitelist gewinnt immer: diese Keys werden künftig behalten.

4) Policy Editor

Whitelist (immer behalten) hat Vorrang vor allem. Blacklist entfernt zusätzlich.

Whitelist (typisch)

token sig signature state expires

Blacklist (typisch)

ref_src source spm

5) Domain Overrides

Domain Overrides erlauben Regeln pro Host (z. B. Amazon, YouTube). Format ist JSON.

{ "amazon.de": { "removeAffiliate": false, "whitelist": ["tag","ref"] }, "youtube.com": { "whitelist": ["v","t"] } }
Tipp: Overrides klein halten. Nur das, was du wirklich brauchst.

6) Redirect/Wrapper entpacken

Manche Links sind Umleitungen (Newsletter/Plattformen). SafeShare Pro kann offline nur entpacken, wenn die Ziel-URL in Parametern steckt (z. B. url=, u=, redirect=).

Hinweis: t.co lässt sich offline meistens nicht entpacken (Server-Redirect).

7) Allowlist-First (Strict)

Strict Mode behält nur Keys aus der Whitelist (plus ggf. Domain-Overrides) und entfernt alles andere. Für Compliance/Standardisierung nützlich – aber kann Links schneller brechen.

Empfehlung: erst mit normalen Defaults starten. Strict erst, wenn du die nötigen Keys pro Domain kennst.

8) Exporte

  • Liste .txt: nur bereinigte URLs.
  • Audit JSON: volle Detaildaten pro Link.
  • Audit CSV: für Sheets/Redaktion/Weitergabe.
  • Policy JSON: versioniert, teilbar (Team/mehrere Geräte).

9) FAQ: Link kaputt?

Was ist der häufigste Grund?

Tokens/Signaturen wurden entfernt (RISK). Lösung: Whitelist-Fix oder manuell whitelisten.

Warum ist ein Link trotz Cleaning noch „lang“?

Weil funktionale Parameter (Suche/Filter/IDs) absichtlich bleiben. SafeShare entfernt primär Tracking-Ballast.

Warum entpackt Unwrap nicht jeden Wrapper?

Offline kann SafeShare nur entpacken, wenn die Ziel-URL als Parameter vorliegt. Server-Redirects brauchen Online-Auflösung.
