Help

SafeShare runs local-first in your browser. If something feels “blocked” or “not working”, it’s usually clipboard permissions, a special link format, or a functional parameter that got removed.

Common issues

1) Paste button doesn’t paste (iOS Safari)

iOS Safari may block programmatic clipboard access. Tap into the input field and use the system Paste menu.

  • Copy a link in any app
  • Open SafeShare
  • Tap into the input field → tap Paste
  • Then press Clean

2) Share button missing

Sharing depends on navigator.share. If not available, SafeShare will still let you copy and paste into WhatsApp/Signal/Mail.

3) “No valid link found”

Some apps wrap links or include extra text. Paste the full message — SafeShare tries to extract the first URL.

If you paste only “www.example.com/…” without https://, SafeShare will try to normalize it, but some edge cases still fail.

If the cleaned link breaks

Sometimes parameters are functional (language, product/session IDs, filters) or intentional (affiliate/ref).

  • Open the original link (without cleaning) to confirm it works.
  • Compare original vs cleaned URL.
  • If only the cleaned version fails, a removed key was likely required.
