Privacy

Last updated: 2026-01-25

SafeShare is built for data minimization. Link cleaning happens locally in your browser (“local-first”).

1. What SafeShare processes

  • You paste or share a URL into the SafeShare web app.
  • The app removes typical tracking parameters (e.g., utm_*, fbclid, gclid) on your device.
  • The cleaned link is shown to you for copying/sharing.

2. What SafeShare does not do

  • No uploads: SafeShare does not send your pasted URLs to a SafeShare server for processing.
  • No link logging: SafeShare does not intentionally store your URLs on the server side.
  • No “invisibility” promise: Cleaning a URL does not prevent tracking on the destination website.
Destination sites may still log IP address, user agent, timestamps, set cookies, and run tracking scripts. SafeShare reduces data you share via URL parameters — it does not block tracking on the page you open.

3. Cookies and storage

SafeShare aims to work without cookies for tracking. The app may use localStorage / sessionStorage for purely local convenience features (for example UI state). These values stay in your browser.

4. Third-party services

This page does not include embedded trackers by design. However, hosting providers and CDNs may process technical request data (e.g., IP address) as part of delivering the website.

If you later add analytics, keep it opt-in and privacy-friendly. Avoid fingerprinting and cross-site identifiers.

5. Your choices

  • Use SafeShare before sharing a link to reduce unnecessary URL parameters.
  • For stronger protection on destination sites: use content blockers, hardened browser settings, and avoid being logged-in where not needed.
  • Prefer privacy-respecting platforms and settings.
