1) App verwenden

  1. App öffnen.
  2. URL in das Feld Eingabe-URL einfügen. Mit ✕ (rechts) lässt sich die Eingabe leeren.
  3. Säubern klicken. Ergebnis erscheint im Feld Bereinigte URL.
  4. Optional:
    • Prüfen: zeigt nur die Änderungen, ohne zu öffnen.
    • Kopieren: Ergebnis in die Zwischenablage.
    • Teilen: Web Share API (wenn verfügbar).
    • Öffnen: Ergebnis in neuem Tab/Fenster.

Unter Änderungen anzeigen siehst du, welche Parameter entfernt/behalten wurden und ob eine Weiterleitung entpackt wurde.

2) Bookmarklet

Das Bookmarklet „SafeShare clean“ säubert die aktuell geöffnete Seite mit einem Klick.

iPhone/iPad (Safari)

  1. bookmarklets.html öffnen und den JavaScript-Code kopieren.
  2. In Safari ein neues Lesezeichen anlegen (Teilen → Lesezeichen).
  3. Lesezeichen bearbeiten → Name: „SafeShare clean“, Adresse/URL: den kopierten javascript:-Code einfügen.

Desktop (Chrome/Firefox/Edge)

  1. bookmarklets.html öffnen.
  2. Den Button/Link „SafeShare clean“ in die Lesezeichenleiste ziehen oder per Rechtsklick als Lesezeichen speichern.

Hinweis: Einige Browser blockieren Drag&Drop von javascript:-Links. In dem Fall Code manuell als Lesezeichen-URL einfügen.

3) Pro aktivieren

  1. Nach dem Kauf leitet Payhip auf pro-activate.html?pro=1 um.
  2. Dadurch wird lokal ss2_pro=1 gesetzt (localStorage).
  3. Zurück zur Startseite → der Badge zeigt „Pro aktiv“.

Deaktivieren: tools.html öffnen → „Pro deaktivieren“ oder localStorage leeren.

4) Bulk-Cleaning

  1. bulk-clean.html öffnen.
  2. Mehrere URLs (eine pro Zeile) einfügen.
  3. Starten → bereinigte Liste kopieren/exportieren.

5) Team-Whitelist / Publisher-Modus

  1. team-setup.html öffnen.
  2. Publisher-Modus aktivieren (optional).
  3. Erlaubte Parameter-Keys eintragen (z. B. utm_source), speichern.

Intern wird ss2_allow im localStorage gepflegt; diese Keys bleiben beim Säubern erhalten.

6) Tools (Service Worker, Caches, localStorage)

tools.html bietet:

  • Service Worker: auflisten, skipWaiting, abmelden.
  • HTTP-Caches: anzeigen, löschen.
  • LocalStorage: anzeigen, gezielt löschen, „Pro aktivieren/deaktivieren“, Hard-Reset.
  • Version/Umgebung: sw.js-Version und Laufzeitinfos.

7) Troubleshooting

Weiße Seite oder alte Inhalte

  1. tools.html → „Alle abmelden“ (SW) → „Alle Caches löschen“.
  2. Seite mit ?nocache=29 neu laden (z. B. /Supersystem/?nocache=29).

Bookmarklet lässt sich nicht ziehen

Browser blockiert javascript:-Drag&Drop. Code auf bookmarklets.html manuell kopieren und als Lesezeichen-URL einfügen.

Pro-Badge fehlt nach Kauf

pro-activate.html?pro=1 aufrufen oder in tools.html „Pro aktivieren“ klicken.

8) Datenschutz

SafeShare arbeitet vollständig lokal im Browser: keine Accounts, keine Server-Logs, keine Cookies, kein Tracking.

Optionale lokale Schlüssel (localStorage):

  • ss2_pro – Pro-Flag (1/fehlend)
  • ss2_allow – Team-Whitelist (kommagetrennte Parameternamen)
  • weitere seitenbezogene UI-Flags nach Bedarf

Details in Datenschutz und Impressum.

9) Rechtliches

Die Nutzung erfolgt auf eigene Verantwortung. Prüfe vor dem Teilen, ob Ziel-URLs korrekt sind. Marken- und Drittinhalte gehören den jeweiligen Eigentümern.
