Pro (professionell)

Pro ist für Social/Redaktion/Publisher/Teams. Es geht nicht nur um “entfernen”, sondern um Policy (Regeln) + Audit (Nachvollziehbarkeit) + Export (Dokumentation).

Pro-Workflow (kurz)

  1. Preset wählen (z. B. “Hygiene” oder “Attribution”)
  2. Link(s) einfügen (einzeln oder als Liste)
  3. Audit prüfen (“Was wurde entfernt?” / “Was blieb?”)
  4. Output kopieren oder exportieren
Pro öffnen Presets verstehen

Presets: was bedeutet das?

Presets sind “Regelpakete”. In Pro kannst du damit schnell zwischen verschiedenen Anforderungen wechseln:

  • Hygiene: Entfernt typische Tracking-Anhänge (für Teilen/Zitate)
  • Attribution: Lässt bewusst bestimmte Parameter drin (für Kampagnen/Publisher)
  • Strict: Entfernt aggressiver (nur wenn du sicher bist)
“Strict” kann Links brechen, wenn funktionale Parameter dabei sind. Nutze Audit + Domain-Overrides.

Audit: „Was wurde geändert?“

Das Audit listet alle Änderungen auf – nicht nur “den ersten Link”. In Pro sollte jede Zeile/URL ein eigenes Ergebnis haben:

  • entfernte Keys (z. B. utm_*, gclid, fbclid …)
  • behaltene Keys (z. B. funktionale Parameter)
  • Regelgrund (Preset/Whitelist/Override)
Wenn du im Audit nichts siehst: dann wurde entweder nichts entfernt, oder dein Preset/Whitelist erlaubt die Keys.

Batch (Listen bereinigen)

  1. Mehrere Links (eine pro Zeile) einfügen
  2. Bereinigen
  3. Ergebnisse pro Zeile prüfen (Audit)

Export (CSV/JSON)

Pro sollte Exporte ermöglichen, damit du Ergebnisse dokumentieren kannst (QA/Redaktion/Team):

  • CSV: gut für Tabellen/Redaktionsplan
  • JSON: gut für technische Weiterverarbeitung
Exporte sind die “Profi-Schere” gegenüber einfachen Cleanern.

Status / Cache / Updates

Wenn du “alte” Versionen siehst oder Elemente fehlen, ist oft der Service Worker Cache aktiv.

Symptome

  • UI wirkt “alt”, obwohl du Code geändert hast
  • Buttons fehlen (z. B. Update-Button in Status)
  • 404/alte Pfade bleiben hängen

Fix (ohne Desktop)

  1. Status öffnen: status.html
  2. Update / Skip Waiting auslösen (falls Button vorhanden)
  3. Seite neu laden
Wenn Status keinen Button zeigt, fehlt wahrscheinlich das UI-Element in status.html. Dann müssen wir status.html komplett setzen (mit Button + SW-Handshake).

FAQ

„Warum entfernt er nur beim ersten Link?“

Dann ist dein Batch-Parser vermutlich auf “erste Zeile” begrenzt. Pro sollte jede Zeile separat verarbeiten.

„Ich sehe nicht, dass utm/ref entfernt wurde.“

Entweder waren die Keys nicht im Entfernen-Set des Presets, oder sie stehen auf einer Whitelist. Pro zeigt im Audit “entfernt” vs “behalten”.

„Pro-Seite 404 wenn ich sie direkt öffne.“

Das ist meistens ein Pfad-/Dateiname-Problem oder Service Worker Cache. Prüfe: exakter Dateiname, Links, und Status/Cache.
