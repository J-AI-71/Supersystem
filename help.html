SafeShare entfernt nur Tracking-Parameter im Link selbst (z. B. utm_*, fbclid, gclid). Es ändert nichts an Browser/Account/IP – es macht dich nicht anonym.

Schnellstart (App)

  1. Link kopieren
  2. In app.html einfügen
  3. Bereinigen → Ergebnis kopieren/teilen
Wenn ein Link nach der Bereinigung nicht funktioniert: 1) Original-Link öffnen (Gegencheck) · 2) bereinigten Link öffnen · 3) wenn nur der bereinigte Link scheitert, war ein Parameter vermutlich funktional → Pro (Audit/Whitelist) oder konservativer bereinigen.

Pro ist ein Offline-Download (nicht als öffentliche Webseite). Starte immer über 01-START.html aus deinem Payhip-Download – so bleibt Pro privat und funktioniert auch ohne Internet.

Typische Probleme – sauber lösen

Link ist nach Bereinigung „kaputt“

  • Vergleich: Original-Link öffnen → bereinigten Link öffnen.
  • Wenn nur der bereinigte Link scheitert: ein Parameter war vermutlich funktional.
  • Pro-Ansatz: Preset konservativ + Audit prüfen + Schlüssel whitelisten (z. B. id, q, page, lang).
Häufig „funktional“/kritisch: token, sig, signature, expires, state, nonce.

Affiliate/Partner-Attribution fehlt

  • Wenn du Attribution behalten willst: nutze Pro + Whitelist.
  • Typische Keys: ref, tag, affid, partner.
  • Dann Link neu bereinigen und Audit prüfen.
Die App entfernt bewusst utm_*. Wenn du UTM bewusst behalten willst, ist das ein Pro-Case (Whitelist/Policy).

404 bei Pro

  • Pro läuft als Offline-Paket. Öffne Pro über 01-START.html im Payhip-Download.
  • Wenn du Dateien umbenannt hast: Links in 01-START.html im Paket prüfen.

„Es passiert nichts“ / Ergebnis wirkt falsch

  • Link neu kopieren (nicht aus Autovervollständigung).
  • Seite neu laden, dann erneut testen.
  • Wenn möglich: in einem privaten Tab testen.
Wenn du magst: schick eine Beispiel-URL (ohne private Daten) + was du erwartest. Dann kann man das Muster schnell erkennen.

iPad / iPhone Hinweise

  • Stabiler Standard-Workflow: App öffnen → Link einfügen → Ergebnis kopieren → teilen.
  • Bookmarklets können auf iOS je nach Browser/Version unzuverlässig sein. Wenn’s hakt: Copy/Paste in der App ist stabil.
Optional später: iOS Shortcut (Share Sheet → Clean with SafeShare) als besonders schneller Workflow.
