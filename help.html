Hilfe & Anleitung

SafeShare säubert Links lokal im Browser: Tracking-Parameter entfernen, Redirects entpacken, klare Kurz-URLs. Keine Konten, keine Server-Logs. Diese Seite führt durch alle Kernfunktionen.

1) App

  1. URL in das Feld einfügen oder per Bookmarklet übergeben.
  2. Option „Säubern + öffnen“ für direkten Zielaufruf, oder „Kopieren“ für die bereinigte URL.
  3. Optional: erlaubte Parameter aus der Team-Whitelist bleiben erhalten.

Unterstützt gängige Redirects (Klarziel wird erkannt) und entfernt typische Tracking-Signaturen (utm_*, gclid, fbclid u. a.).

Zur App

2) Bookmarklets

Desktop

  • Lesezeichenleiste einblenden.
  • Auf bookmarklets.html den gewünschten Button in die Leiste ziehen.
  • Auf einer Zielseite klicken → Link wird an SafeShare übergeben.

Mobil (iOS/Android)

  • Neues Lesezeichen anlegen, Namen vergeben (z. B. „SafeShare Clean“).
  • URL des Lesezeichens durch den JavaScript-Code ersetzen (auf bookmarklets.html „Code kopieren“).

3) Team-Whitelist

Parameter definieren, die erhalten bleiben dürfen (z. B. tag, ref, aff_id). Gilt lokal im Browser für App, Bookmarklets und Bulk-Clean.

  1. Team-Setup öffnen.
  2. Neue Parameternamen hinzufügen (klein geschrieben, a–z, 0–9, _ - .).
  3. Export (JSON) für Team-Doku optional speichern.
Team-Whitelist öffnen

4) Bulk-Clean (Pro)

Viele URLs in einem Rutsch säubern; Ergebnis als Liste oder CSV exportieren.

  1. Liste/Zeilen einfügen (eine URL pro Zeile; extrahiert auch aus Fließtext).
  2. Optionen wählen: Redirects entpacken, Whitelist beibehalten, Duplikate entfernen.
  3. „Säubern“ → „CSV exportieren“ oder „Ergebnis kopieren“.
Bulk-Clean öffnen

5) Redirects entpacken

Übliche Weiterleitungen (z. B. google.com/url, t.co, l.facebook.com) werden zum Klarziel aufgelöst. Optional Tracking entfernen und Whitelist anwenden.

Redirect-Tool öffnen Bookmarklet „Redirect entpacken“

6) PWA & Offline

  • iOS (Safari): Seite öffnen → Teilen → „Zum Home-Bildschirm“.
  • Android (Chrome): Menü → „App installieren“ / „Zum Startbildschirm“.
  • Offline: Nach Installation sind Kernseiten offline nutzbar.

Bei Layoutproblemen hart neu laden: URL mit ?nocache=YYYYMMDDHH öffnen oder Status → „Auf Updates prüfen“.

7) FAQ (Kurz)

Erfordert SafeShare ein Konto?

Nein. Es gibt keine Accounts und keine SafeShare-Server-Logs.

Wie wird die Whitelist gespeichert?

Lokal im Browser (localStorage) unter dem Schlüssel ss_whitelist.

Welche Parameter werden entfernt?

Typische Tracking-Signaturen wie utm_*, gclid, fbclid, msclkid u. a.

Gibt es eine 1-Seiten-Anleitung?

Ja, Quickstart mit QR-Code zur App.

Troubleshooting

App lädt, aber Buttons reagieren nicht

  • Cache/Service Worker: Status öffnen → „Auf Updates prüfen“ → „Wartende Version aktivieren“.
  • Harter Reload: URL mit ?nocache=1 aufrufen (z. B. app.html?nocache=1).
  • CSP prüfen: Im <head> muss script-src 'self' stehen; für Netztests connect-src 'self' https:.

Bookmarklet lässt sich nicht ziehen (iOS/Safari)

  • Statt Drag&Drop: Langdruck → „Zu Favoriten“ oder „Lesezeichen hinzufügen“.
  • Alternativ: Code kopieren und manuell als Lesezeichen-URL einfügen.

Offline-Nutzung funktioniert nicht

  • Seite einmal online öffnen, damit der Service Worker die App cached.
  • offline.html ist Fallback; prüfe in sw.js PRECACHE_URLS und OFFLINE_URL.

„Säubern“ entfernt gewünschte Parameter

  • Whitelist im Team-Setup pflegen (z. B. tag, ref). Die dort eingetragenen Keys bleiben erhalten.

Redirect wird nicht entpackt

  • Parameter-basierte Weiterleitungen (z. B. l.facebook.com/l.php?u=) funktionieren offline.
  • Shortener wie t.co benötigen Internet (echte HTTP-Auflösung).

Kopieren/Teilen funktioniert nicht

  • Clipboard/Share-APIs sind abhängig vom Browser. Fallback: Ergebnis manuell markieren und kopieren.

8) Troubleshooting

  • Bookmarklet lässt sich nicht ziehen: Per Rechtsklick „Link zu Lesezeichen hinzufügen“ oder nur den Code kopieren und als Lesezeichen-URL einsetzen.
  • Button „Säubern + öffnen“ reagiert nicht: Pop-up-Blocker prüfen oder Ziel in neuem Tab erlauben.
  • Änderungen erscheinen nicht: Seite mit ?nocache=TIMESTAMP öffnen oder in Status „Auf Updates prüfen“.
  • Whitelist greift nicht: Schreibweise (klein) prüfen; Seite ggf. neu laden.
